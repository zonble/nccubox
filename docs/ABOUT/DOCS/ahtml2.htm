「盒子．道南生活網路誌」（以下簡稱「盒子」）可以說是我在政大 就學四年期間最重要的作品，特色在於網頁「浮動檢視工具列」、「 浮動期數索引」等界面的創新、網路多媒體電子雜誌的多方面嘗試、 以及就我個人經驗而言，像「盒子」這樣的網站幾乎不可能是一個人 獨立完成，我到現在已經汲汲營營了半年之久。

「盒子」對我個人而言的重要性在於：一方面這是一份以全球資訊網 為介面的多媒體網路月刊，當中所有的文字（校園評論、小說故事、 笑話等）、圖形（圖示與影像設計）及聲音（ MIDI ）全部由個人創 作，每個月幾乎要花上十五到二十天工作。二方面是個人理想的落實 ，也是我透過實作的方式，為從事網頁設計經驗中曾經歷過的難題提 出解答。

內容與設計概念簡介

「盒子」是我所參與「伺服器玖玖」站台／計畫中的一環，位在「伺 服器玖玖」主機中，網址http://server99.dormA.nccu.edu.tw/BOX ，八十六年三月開始籌備、四月創刊，以政大師生為目標讀者。每期 內容包含：編輯室報告、當月主題、盒子觀點及附贈的音樂創作及桌 面背景底圖。

內容取向最初是強調不流俗的娛樂趣味，後來同時也希望兼具深沉反 省的人文精神，所以文章中許多是以誇張筆調譏諷政大校園的事件與 文化（如第四期的〈原來他們怕的是光環〉），也有帶有感情的散文 （如第二期的〈失去一種觀視的角度〉）。在「盒子」中同時可以找 到無釐頭搞笑與深沈批判。

刊名為什麼叫做「盒子」？就像在每期首頁都會強調的：「我們想把 道南橋這端的林林總總，放進一個盒子裡，所以，這就是我們的盒子 。」盒子的器物功能是裝盛，所以我希望以「盒子」裝滿政大的資訊 ，當政大師生想要得到任何資訊時，只需要「打開」盒子。

同樣是提供資訊，政大校園內已經有兩三份重要的校園新聞刊物：刊 載全國大學校園新聞的新聞系實習報紙「大學報」、提供學生會消息 的「政大學生會訊」以及已有三十年歷史的「自強報」（自強報同時 是「伺服器玖玖」中的網刊之一）。我便定位「盒子」為雜誌，以取 向上與新聞刊物間做區隔。我在「盒子」中的「掀起了你的盒子來」 中也強調過這一點。

因為強調趣味性，設計便採用明亮的用色。每期的色彩計畫都以純色 為主，且都是以色相中的二次色、三次色為主色，歡迎畫面中的使用 青色、前兩期採用鮮黃、第四期是純白與粉紅粉藍搭配、第五期黃藍 互補色運用及第六期使用的亮紫。

盒子的精神

當初刊行「盒子」，或是架設「伺服器玖玖」網站，個人都竊自認為 那充滿相當的理想性。「伺服器玖玖」是要「救救」政大匱乏的網路 資源與文化，而實際的落實便是像「盒子」這樣的網路刊物。整個盒 子精神的陳述在「盒子」的「盒子超文本宣言」中描述得更為詳細。

說到提昇校園的網路資源，「盒子」與浩然圖書館、清罻園等大型計 畫提供的豐富學術研究相較實在不算什麼，但我個人更為關心的是： 我是否能提供給在我生活周遭、在同一個校園息息相關的人們切身的 資訊，如何讓他們感受到我在字裡行間對這個校園所投注的感情、以 及對這樣的校園文化／網路文化的警訊？這麼說來，我似乎做的，就 是以網路作為社區報的工作。

網路上的文字

在網路熱潮蔓燒的今日， 設計者幾乎全在關心如何不斷使用 Java、 ActiveX 以及 Dynamic HTML 設計動態網頁吸引注意，而我在「盒子 」中更關心的似乎是一個不受重視的領域，就是「網路上的文字應該 是什麼樣子？」、或是「什麼是網路文字的本質」？

再進一步具體描述，就是在一份網路文件當中最好多長？超過多少字 的文件讀者就難以吸收？每行應該要有多少字？超過多少字讀者就不 易轉行？字級最好是多大、字型最好是哪一種，在螢幕上看起來最美 觀？看起來最不吃力？長篇的文件又應該如何處理？

回顧新聞史我們可以發現，現代報業在本世紀初出現之後，為了因應 編輯型態而造就出至目前為止仍顛仆不破的「倒金字塔」新聞寫作， 而在世紀末網際網路全球資訊網興起之後，是否又會因此出現新的文 體，好方便使用者／讀者的資訊認知？

另外目前國內外關於超文本的研究指出，超文本的跳躍性閱讀最適合 用於讓讀者找到一篇文件中的一段短而獨立的章節。但是從文學上來 說，跳躍性的文學閱讀在傳統上根本不存在，而在跳躍性閱讀逐漸成 為主流之後，文學創作又應該如何因應？而「跳躍性的文學」又是否 可以稱的上是文學？

在「盒子」的設計中，目前能夠解答上述問題者還相當少，最多只是 找到了一個視覺上還不錯的文件寬度。「盒子」每篇文章都使用超文 件表格語法（ Table ）將寬度限制在四百八十像素（ Pixel ）之內 ，並且將這一表格置於文件的正中央，內文是十二級細明體。

內文的大小可說適中，因為更小的中文字在螢幕上顯示時筆畫會糾結 在一起，而每行的字數大約是二十五字，周圍留白，整個頁面基本設 計有點像用微軟文書處理器 Word 打好印出的文件，在 640x480、 800x600 解析度螢幕上看起來都算不錯。 加上一個窗框（ Frame ） 後，在 800x600 螢幕上顯示更加美觀，在 640x480 螢幕上使用者也 不須移動滑鼠就可以看到整行內文。

應用程式化的窗框界面

「盒子」界面中最特別的應該是「浮動檢視工具列」以及「浮動期數 索引」的設計，而這樣的設計乃是根源於我的「超文件端瀏覽器介面 」的構想， 也就是「利用超文件中的窗框、JavaScript 彈跳式視窗 等功能，替代原先的瀏覽器功能，增加窗框內連結的功能性。」

在超文件的界面設計中，即使畫面、內容的設計如何具有風格，但是 還是有地方是設計者無法更動的，就是瀏覽器的界面：選單以及工具 按鈕的排列等。一方面使得設計者可以設計的部份受到侷限，二方面 原先的瀏覽器設計並不能符合所有瀏覽文件的需要，而造成瀏覽的困 擾。

例以一般瀏覽器都有的「前進」與「後退」功能：「前進」都是向右 的按鈕、「後退」都是向左的按鈕，這麼設計的原因是英文或是歐洲 文字都是從左邊往右邊寫。但是假使要設計一個由上往下讀的中文網 站，那「前進」就應該「往下」、「後退」就應該「往上」才是，才 能符合瀏覽的需要。

也就是說，每一篇網頁因為其不同的瀏覽特性，而更動使用者的瀏覽 器外觀以增進讀者的認知，同時也可以因此更徹底的讓使用者感受到 站台的風格。

而「浮動工具列」就是「加強超文件中的功能性」的初步嘗試。這個 設計也是為了解決使用者螢幕太小的問題：在許多網站都要求使用者 螢幕解析度在 800x600 之際，使用 640x480 解析度的人卻不在少數 ， 「盒子」最初的設計可以在 640x480 順利瀏覽，但是再決定使用 窗框後就顯得畫面擁擠，最後我便以「浮動工具列」代替原先窗框的 功能解決這個問題。

盒子今後的發展

經歷半年之後，「盒子」也已經發行了五六期，與許多知名的國內學 生自製的網路期刊相較還不算太差，如「大便報」從八十三年創刊後 到現在發行八期、「烘培姬」大概也是五六期。個人以為「盒子」的 成績單還不算太差。

大四了，該是為自己的人生加油的時候。明年這個時候我不會繼續出 現在政大校園，在一年之內我勢必會停止「盒子」的編輯工作，未來 會停刊、還是有來者願意接替「盒子」的編輯工作？我現在還沒有具 體的計畫。不過最近，我還是會繼續做下去。

只是因為想把道南橋頭這端的林林總總，放進一個盒子裡，所以，這 就是「盒子」、就是「盒子．道南生活網路誌」。
